In this practical we will see how to remove the ‘HTTPS’ warning error that you get whenever you try
to connect to your server. s a reminder, we are getting this error as I don’t own ucl.ac.uk so can’t
ask for a certificate to be issued on that domain.
Instructions are primarily for Windows users, but draft (untested) instructions for Apple users are
given below.
** NB: The instructions in this practical should be carried out with caution and
are at your own risk. Never use these instructions for a certificate that you
don’t trust and/or where you don’t know the source **
Every time you connect to https://<<your server IP address>> the server sends a certificate to your
browser. Your browser then compares this certificate to a list of official certificates to see if it is
legitimate/registered. If it is, then the browser knows to trust your server.
Even though we don’t have a registered certificate, we can manually tell our machine to trust the
certificate - i.e. tell the machine that this is a trusted website. This is something you might need to
do when you are developing HTTPS code on internal servers.
The way this is done is by adding the certificate to a list of trusted certificates held on the machine.
Instructions for Windows Users
Step 1 - Convert the Certificate to PKCS format (PKCS = public key cryptography standard). This will
allow it to be recognised by the Windows machine.
1. Open Chrome and go to: https://<<your server IP address>>
2. Click on the ‘not secure button’ to get details of the certificate
3. Click on the DETAILS tab
4. Follow the certificate export wizard steps (click COPY TO FILE to start the process)
5. Add the certificate to the trusted certificates list on your computer by typing (from the
search/Cortana box) manage computer certificates
6. Add the certificate under THIRD PARTY ROOT CERTIFICATION AUTHORITIES
7. Click IMPORT
8. Clear the CHROME cache
9. Restart your laptop/desktop machine
10. Retype https://<<your server IP address>> into Chrome. You should now see a padlock
as follows:
11. If you click on the padlock and then click on CERTIFICATE you will see the validity dates
of the certificate (NB the details might be different)
Instructions for Apple Users (Untested)
** NB: These instructions were created on an iPad and are not tested. Please share any connections
on Moodle **
1. Download the developer5.p7b certificate from Moodle
2. If necessary, e-mail the certificate to yourself
3. Once you receive the e-mail, double-click on the certificate to start installation
4. Click INSTALL NOW. You will probably be asked for your PIN or password
5. Click INSTALL (top right corner)
6. Click INSTALL again
7. Clear browsing history e.g. in Safari
8. Browse to https://<<your server IP address>> and you should now see a secure lock on the
site.
